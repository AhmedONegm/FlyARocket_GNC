This file contains some of the primary, "every-day" functions used by the GNC algorithm to achieve its tasks. The file is called "master_funcs" because the functions here are general in that they tend to appear everywhere in the code to perform routine tasks like file I/O, waiting, etc.

This function, given a wanted thrust, assigns the required PWM to produce that thrust given the PWM_valve_charac[] and R_valve_charac[] arrays which assigns the correct PWM for a given thrust level. Typical thrust curves can be seen on the first figure at page 2 of datasheet found here. However, we manually measured the thrust level for a given PWM using a balance. The data was collected into a spreadsheet and the following graph was produced:

The valves in our application are controlled in open-loop due to space and time constraints on implementing sensors to close the loop on valve control. This is suboptimal, of course, due to valves heating up, cooling down, hysteresis, etc. that would slightly make the thrust curve change during flight.

Parameters
R1_thrustThe thrust we want the valve R1 to output.
R2_thrustThe thrust we want the valve R2 to output.
R3_thrustThe thrust we want the valve R3 to output.
R4_thrustThe thrust we want the valve R4 to output.
pwm1The pointer to the PWM1 value (for valve R1).
pwm2The pointer to the PWM2 value (for valve R2).
pwm3The pointer to the PWM3 value (for valve R3).
pwm4The pointer to the PWM4 value (for valve R4).
